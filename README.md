# Jasace ACE — Website & CMS

Professional website for Jasace ACE, a Las Vegas-based architecture, construction, and engineering consulting firm.

**Live site:** [jasace-site.onrender.com](https://jasace-site.onrender.com)  
**Admin panel:** [jasace-site.onrender.com/admin](https://jasace-site.onrender.com/admin)

---

## Overview

The site is built with **Next.js 14** and **Tailwind CSS**, with a custom admin panel backed by **Convex** for content management. Jason (or anyone on the team) can update all site content — blog posts, projects, team bios, testimonials, images, and more — without touching code.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 14 (App Router) |
| Styling | Tailwind CSS |
| Backend / CMS | Convex (convex.dev) |
| Rich Text Editor | TipTap |
| Hosting | Render (portable — works on any Node.js host) |

## Project Structure

```
src/
├── app/              # Pages (home, about, services, blog, contact, admin)
├── components/       # Reusable UI components
├── data/             # Fallback/default content (used when CMS is loading)
├── lib/              # Utilities, CMS helpers, admin auth
└── types/            # TypeScript interfaces

convex/
├── schema.ts         # Database tables and field definitions
├── *.ts              # Query and mutation functions for each content type
└── seed.ts           # Seeds the database with initial content
```

## Content Management

The admin panel at `/admin` lets you manage:

- **Hero** — homepage headline, background image, CTAs
- **About** — company story, philosophy, team highlights
- **Services** — architecture, construction, engineering descriptions
- **Projects** — case studies with specs, galleries, and details
- **Blog** — full rich text editor (bold, headings, lists, links, blockquotes)
- **Team** — member bios and photos
- **Testimonials** — client quotes
- **Stats** — the "By the Numbers" section
- **Settings** — company info, social links, footer
- **Subscribers** — view and export newsletter signups (CSV)

All content updates are **instant** — no redeploy needed. Images can be uploaded via drag-and-drop or pasted as URLs.

## Getting Started

### Prerequisites

- Node.js 18+
- A Convex account (free at [convex.dev](https://convex.dev))

### Setup

```bash
# Install dependencies
npm install

# Set up Convex (creates project + .env.local)
npx convex dev --once

# Seed the database with default content
npx convex run seed:seedDatabase

# Start the dev server
npm run dev
```

The site will be running at `http://localhost:3000`.

### Environment Variables

Create a `.env.local` file (auto-generated by `npx convex dev`):

```
CONVEX_DEPLOYMENT=dev:your-deployment-name
NEXT_PUBLIC_CONVEX_URL=https://your-deployment-name.convex.cloud
NEXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment-name.convex.site
ADMIN_PASSWORD=Jasace2026!
```

## Deployment

### Build & Run

```bash
npm run build    # Builds the Next.js app
npm start        # Starts the production server
```

### Deploy Convex to Production

```bash
npx convex deploy
```

This pushes the schema and functions to the production Convex deployment.

### Hosting

The site is a **standard Next.js app** with no platform-specific dependencies. It runs on any Node.js hosting:

- **Render** (current) — just connect the repo and set env vars
- **Vercel** — native Next.js support, zero config
- **AWS** (Amplify, EC2, ECS) — `npm run build` + `npm start`
- **Self-hosted** — any server with Node.js 18+

### External Dependencies

| Service | Purpose | Required? |
|---------|---------|-----------|
| Convex | Database, real-time queries, file storage | Yes |
| Google Fonts | Typography (Syne, Archivo, etc.) | Yes (loaded via next/font) |

No other external APIs or services are required.
